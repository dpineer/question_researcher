# AI 互动助教 - 双引擎互动出题与学情分析系统

一个基于 Flutter 的桌面端 AI 助教系统，集成了本地模型与云端大模型，支持混合题型生成、智能批改、学情分析和知识库编辑功能。

## 🚀 核心功能

### 1. 双引擎出题系统
- **本地模型引擎**：使用 LM-Studio 本地模型进行初步内容提炼和重排序
- **云端大模型引擎**：使用 DeepSeek API 进行高质量混合题型生成
- **智能题型分布**：自动生成单选、多选、填空、简答等多种题型
- **语义检索优化**：基于向量嵌入的语义检索，提取最相关内容
- **重排序功能**：优化内容结构，提升出题质量

### 2. 智能学情分析
- **AI 实时批改**：客观题自动评分，主观题 AI 诊断
- **个性化指导**：基于历史错题生成针对性学习建议
- **答题历史追踪**：完整记录每次测验的详细数据
- **形成性评价**：针对每个回答提供详细的诊断反馈

### 3. 知识库管理
- **多格式导入**：支持文本、图片、PDF、Word 等多种格式
- **OCR 识别**：内置视觉模型自动提取图片和扫描件中的文字
- **知识库编辑**：可随时编辑原始参考资料，支持 DAG 图生成辅助理解
- **批量文件处理**：支持多文件批量导入和自动解析

### 4. 新增高级功能
- **知识库编辑界面**：新增 `KnowledgeEditScreen` 组件，支持编辑已保存的知识库
- **DAG 图生成**：利用 AI 将复杂文本转换为 Mermaid 格式的有向无环图，辅助理解知识结构
- **语义检索服务**：基于向量嵌入的语义检索，智能提取相关内容
- **实时日志系统**：内置业务日志总线，实时监控处理进度
- **多模态模型集成**：支持视觉模型、重排序模型、嵌入模型等多种本地模型

## 🛠️ 技术架构

### 前端框架
- **Flutter 3.x**：跨平台桌面应用开发
- **Material Design 3**：现代化 UI 设计
- **Provider**：状态管理
- **Markdown 渲染**：支持 LaTeX 公式渲染的 Markdown 显示

### 后端服务
- **SQLite**：本地数据存储（支持版本迁移）
- **Dio**：HTTP 客户端
- **sqflite_common_ffi**：桌面端 SQLite 支持
- **文件系统操作**：支持多平台文件操作

### AI 集成
- **DeepSeek API**：云端大模型服务（题目生成和诊断）
- **LM-Studio**：本地模型服务（支持多种模型）
- **OCR 视觉模型**：图片文字识别
- **向量嵌入模型**：语义检索和相似度计算
- **重排序模型**：内容优化和结构重排

### 文件处理
- **FilePicker**：多格式文件选择
- **PDF 解析**：原生 PDF 引擎支持（Linux 需要 poppler-utils）
- **图片处理**：Base64 编码与传输
- **多模态处理**：支持图片、PDF 的视觉识别

## 📦 安装与运行

### 环境要求
- Flutter 3.0+
- Dart 3.0+
- Linux 桌面环境（支持 Windows/macOS）
- 可选：LM-Studio 本地模型服务

### 安装步骤
```bash
# 克隆项目
git clone https://github.com/dpineer/question_researcher.git
cd question_researcher

# 安装依赖
flutter pub get

# 运行应用（Linux 桌面）
flutter run -d linux --debug
```

### Linux 系统依赖
```bash
# 安装 PDF 处理工具（用于原生 PDF 解析）
sudo apt-get install poppler-utils
```

### 配置说明
首次运行前需要在设置页面配置：

1. **DeepSeek API Key**：必需，用于云端大模型服务（题目生成和诊断）
2. **LM-Studio URL**：本地模型服务地址（默认：http://localhost:1234/v1）
3. **模型标识符配置**：
   - 文本提炼模型（Chat Model ID）
   - 视觉解析模型（Vision Model ID）
   - 重排序模型（Rerank Model ID，可选）
   - 嵌入模型（Embedding Model ID，可选）
4. **功能开关**：启用/禁用重排序功能

## 🎯 使用指南

### 1. 创建题库
1. 点击首页右下角"导入资料制卷"按钮
2. 输入出题方向（如"计算机网络基础"）
3. 导入知识资料（支持文本粘贴或文件批量导入）
4. 设置题量和难度，点击"双流引擎开始生成"
5. 实时查看处理进度和日志信息

### 2. 进行测验
1. 在首页题库列表中选择要测试的题库
2. 按顺序答题，支持单选、多选、填空、简答
3. 答题界面支持 Markdown 和 LaTeX 公式渲染
4. 完成所有题目后提交评卷

### 3. 查看分析报告
1. 提交后自动跳转到 AI 诊断报告页面
2. 查看每道题的详细解析和 AI 诊断
3. 可查看历史答题记录和个性化指导
4. 基于历史错题生成个性化学习建议

### 4. 编辑知识库
1. 在首页题库列表中点击"编辑原始知识库"按钮（绿色编辑图标）
2. 进入编辑页面修改原始参考资料
3. 支持追加导入新文件到知识库
4. 点击"保存并覆盖"保存修改
5. 点击"AI 辅助理解：生成逻辑 DAG 图"按钮，生成知识结构图

### 5. 查看历史记录
1. 在首页题库列表中点击"查看历史记录"按钮
2. 查看该题库的所有答题历史
3. 基于历史记录生成个性化学习指导

## 🔧 系统架构详解

### 数据层架构
- **DatabaseHelper**：SQLite 数据库操作，支持版本迁移
- **SavedExam**：题库实体，包含题目 JSON 和原始知识库
- **ExamQuestion**：题目实体，支持多种题型
- **ExamRecord**：答题记录实体，包含用户答案和 AI 反馈

### 服务层架构
- **DualAIService**：双模型 AI 服务，集成本地和云端模型
- **SemanticRetrievalService**：语义检索服务，基于向量嵌入
- **ConfigService**：配置管理服务，支持持久化存储

### 状态管理架构
- **ExamProvider**：出题状态管理，包含进度跟踪
- **ExamTakingProvider**：答题状态管理，包含答案存储
- **ThemeProvider**：主题管理，支持明暗模式切换

### 视图层架构
- **HomeScreen**：首页，显示题库列表
- **KnowledgeInputScreen**：知识导入页面，支持多格式导入
- **ExamTakingScreen**：答题页面，支持多种题型输入
- **ExamResultScreen**：结果页面，显示详细诊断
- **ExamHistoryScreen**：历史记录页面
- **SettingsScreen**：设置页面，配置模型参数
- **KnowledgeEditScreen**：知识库编辑页面

## ⚙️ 配置参数详解

### 云端模型配置
- **DeepSeek API Key**：必需，用于高质量题目生成和诊断
- **模型**：deepseek-chat
- **超时设置**：支持长文本处理的超时配置

### 本地模型配置
- **LM-Studio URL**：本地模型服务地址
- **文本提炼模型**：用于内容提炼和考点提取
- **视觉解析模型**：用于图片和 PDF 的文字识别
- **重排序模型**：用于内容结构优化（可选）
- **嵌入模型**：用于语义检索和向量计算（可选）

- **启用重排序**：优化内容结构，提升出题质量
- **启用 OCR**：启用图片和 PDF 的文字识别
- **启用原生 PDF 解析**：使用系统工具解析 PDF（Linux）

## 🐛 故障排除

### 常见问题

1. **API Key 未配置**：在设置页面配置 DeepSeek API Key
2. **本地模型连接失败**：
   - 检查 LM-Studio 是否运行在 http://localhost:1234
   - 确认模型标识符与 LM-Studio 中加载的模型一致
3. **数据库迁移问题**：删除 `~/.ai_teacher/exams_v2.db` 重新创建
4. **PDF 解析失败**：
   - Linux 系统需要安装 poppler-utils：`sudo apt-get install poppler-utils`
   - 检查文件权限和路径
5. **图片识别失败**：
   - 确认视觉模型已正确配置
   - 检查图片格式和大小
6. **进度卡住**：
   - 检查网络连接
   - 查看实时日志获取详细信息
   - 调整超时设置

### 日志查看
应用内置实时日志系统，具有以下特性：
- **彩色日志显示**：不同级别日志使用不同颜色
- **实时更新**：处理进度实时显示
- **文件处理进度**：显示批量文件处理进度
- **Token 消耗统计**：显示 AI 模型的 Token 使用情况

可在知识导入页面底部展开日志面板查看详细运行日志。

## 🔄 工作流程

### 出题流程
1. 用户导入知识资料（文本/图片/PDF）
2. 系统进行预处理（OCR 识别、文本提取）
3. 语义检索提取相关内容
4. 重排序优化内容结构
5. 本地模型提炼考点
6. 云端模型生成混合题型
7. 保存到数据库并显示

### 答题流程
1. 用户选择题库开始测验
2. 按顺序回答各种题型
3. 系统记录用户答案
4. 提交后系统进行批改
5. 客观题自动评分，主观题 AI 诊断
6. 生成详细诊断报告
7. 保存答题记录

### 分析流程
1. 收集历史答题记录
2. 分析错题模式和薄弱点
3. 生成个性化学习指导
4. 提供针对性改进建议
